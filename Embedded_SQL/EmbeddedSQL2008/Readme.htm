JDBC - MS_SQL Server 2008   Interface

Software installation:
  • Follow all the instructions for SQL Server related installation (in the given sequence).

  • If Eclipse is not already installed,
    download it from http://www.eclipse.org 
    ["downloads" tab and click on Eclipse SDK 3.1.2, Windows (103 MB)]
    and install it.

SQL Server 2008 Setup:

  • Set mixed authentication mode:
    Start -> All Programs -> Microsoft SQL Server 2008 -> SQL Server Management Studio
    On the "Connect to Server" window, login with the valid username and password;  Click "Connect"
    At the top of the left panel, right click on machine(SQL Server 10.0. ...) and select Properties
    On the left panel of the "Server Properties" window, select Security
    Choose "SQL Server and Windows Authentication mode" and click OK

  • Create a new database:
    On the left panel, expand "Databases"
    Right click "Databases"  -> Select "New Database..." and create an empty "company" database

  • Populate the new database:
    On the left panel, right click on "company" database and select "New Query"
      Click on "Open File" icon (from the icon bar)
      Browse and choose setupCompany.sql -> Open;
    On the icon bar, a box on the left side displays the name of the active database.
    Click the dropdown list of databases of that box and select "company" database.
    Click "! Execute" icon

  • Create a new user and set permissions:
    On the left panel of Server Management Studio window,
    Click "Security" and right click on "New" and select "Login..."
      General: Name as Application program username(cgs4366), Choose SQL Server Authentication,
          Password(4366), Confirm password (4366), company as default database
      UserMapping: Enable company (top), Enable db_datareader, db_datawriter, public (bottom)
      Click OK

  • NetBeans Configuration:
    Start a new project and paste all five sample Java programs into the default source package
    RightClick on Libraries => Add JAR/Fodler... => choose jtds-1.2.4.jar file from the expanded downloaded folder.
    Run each sample program (remember to edit database server hostname in the sample program file as needed).
    If the project has any reference error, resolve all reference errors and then perform a clean build.

  • Eclipse Configuration:
    Configure Eclipse with these configuration steps
    Use jtds-1.2.4.jar instead of sqljdbc.jar in the above eclipse configuration steps.

Testing Database Connectivity:

  • To establish a connection between a Java database application program and

    • Microsoft SQL server, the application program must perform the following two steps:
      - register appropriate database driver (JDBC)
      - connect to a database server with appropriate authentication

  • Following two sample programs test database connectivity

    •    TestDB1.java   -- fixed username & password
       TestDB2.java   -- tests for any username & password

Testing Dynamic SQL queries:

  • JDBC - MS_SQL can support dynamic queries.
  • Following three sample programs demonstrate dynamic queries and assume the default database is "company".

    •    TestDB3.java   -- fixed dynamic query string
       TestDB4.java   -- dynamic query string with an argument
       TestDB5.java   -- nested dynamic queries
